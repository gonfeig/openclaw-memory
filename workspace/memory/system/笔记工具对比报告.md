## 五、OpenClaw 接入能力

| 平台 | OpenClaw Skill | API 类型 | 接入难度 | 已验证 |
|------|--------------|---------|---------|-------|
| **IMA 笔记** | ✅ `ima-skill`（workspace） | IMA OpenAPI | ⭐ 简单 | ✅ 已在用 |
| **腾讯文档** | ✅ `tencent-docs`（workspace） | MCP | ⭐ 简单 | ✅ 已在用 |
| **飞书云文档** | ✅ `feishu-doc/wiki/drive`（workspace） | 飞书 OpenAPI | ⭐ 简单 | ✅ 已在用 |
| **Obsidian** | ✅ `obsidian`（workspace） | Local REST API 插件 | ⭐⭐ 中等（需桌面运行） | ⚠️ 待验证 |
| **Notion** | ❌ 无原生 Skill | Notion API | ⭐⭐⭐ 需自建 | ⚠️ 有方案（clawport.io） |
| **Logseq** | ❌ 无 | ❌ 无官方 API | - | ❌ |
| **语雀** | ❌ 无 Skill | 语雀 OpenAPI | ⭐⭐⭐ 需自建 | ⚠️ API 存在 |

### 接入方案详情

**Obsidian**：需安装 `Local REST API` 社区插件，桌面 App 运行中时可通过 HTTP API 读写笔记。安全性：自签名 HTTPS + API Key。

**Notion**：已有 OpenClaw 集成方案（clawport.io），通过 Notion API 读取数据库和页面，需要 Plus 账号。

**语雀**：有开放 API，但无原生 OpenClaw Skill，需自行封装。

**Logseq**：无 API，只能通过 Git 同步或文件系统访问。

## 九、结论与建议

### gonfeig 场景适配分析

**当前已集成**：IMA 笔记、腾讯文档、飞书云文档（OpenClaw Skill 已有）

| 场景 | 推荐平台 | 理由 |
|------|---------|------|
| 个人 AI 辅助笔记 | **IMA 笔记** | OpenClaw 已集成、DeepSeek 免费、AI 能力强 |
| OpenClaw 自动写入 | **腾讯文档** | Skill 已验证可用 |
| 飞书消息相关 | **飞书云文档** | 飞书生态集成，OpenClaw Skill 完整 |
| 长期知识沉淀 | **Obsidian** | 本地 Markdown，数据完全可控，插件生态最强 |
| 开源免费主义 | **Logseq** | 完全免费，开源，大纲式思考友好 |
| 团队协作场景 | **Notion** | 全球最佳团队协作平台 |
| 国内企业知识库 | **语雀** | 阿里生态，文档结构化能力强 |

### OpenClaw 接入建议优先级

1. **当前已在用**：IMA 笔记 ✅、腾讯文档 ✅、飞书 ✅
2. **值得扩展接入**：Obsidian（Local REST API 最潜力）、Notion（有方案待实现）
3. **暂不优先**：Logseq（无 API）、语雀（无 Skill、生态不熟）

### 最终推荐

对于 gonfeig 的「数字第二大脑」目标：

| 角色 | 推荐工具 | 说明 |
|------|---------|------|
| **主力笔记** | **IMA 笔记** | OpenClaw 原生集成，AI 最强，零成本 |
| **备份/归档** | **Obsidian** | 本地 Markdown，数据永不丢失，可配合 Git |
| **OpenClaw 写入** | **腾讯文档** | Skill 成熟，已验证可用 |
| **长期知识管理** | **Notion**（可选） | 未来团队扩展时考虑 |

> IMA 笔记 + Obsidian + 腾讯文档的组合覆盖了 AI 能力、本地可控性、OpenClaw 自动化三个核心需求，是当前最优解。
