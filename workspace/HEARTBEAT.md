# HEARTBEAT.md - 定时自检 + 主动建议

## 规则
- 工作时间（08:00 - 23:00）每 2-3 小时做一次轻量巡检
- 非工作时间（23:00 - 08:00）静默，有紧急情况才唤醒
- 每次巡检不重复做同样的事，轮换检查项目
- 超过 30 分钟的巡检任务，降级到 cron 处理

---

## 轮换检查项

### 检查 1：定时任务状态（每 6 小时）
- 扫描所有 cron 任务，标记 error 状态
- 如有 error，按「故障处理规范」处理（最多 3 次重试）
- 无问题则 HEARTBEAT_OK

### 检查 2：Git 推送状态（每 12 小时）
- 检查 openclaw-memory 仓库是否有未推送的 commit
- 如有_pending > 3，触发一次 push（自动执行，不打扰）
- 无则 HEARTBEAT_OK

### 检查 3：存储空间（每天一次）
- 检查 /root/.openclaw/ 磁盘使用率
- 超过 80% 告警，通知用户
- 未超过则 HEARTBEAT_OK

### 检查 4：记忆新鲜度 + 主动建议（每天一次）
- 检查 scene_blocks 是否有超过 7 天未更新的高频场景（热度 > 10）
- 检查 MEMORY.md 是否有待跟进事项超时（超 7 天无更新）
- **主动建议**：如有「待跟进」超过 14 天，在巡检时简短提醒用户
- 无则 HEARTBEAT_OK

### 检查 5：今日事件摘要（每天 20:00）
- 整理今日重要对话摘要
- 检查是否有未完成的用户请求
- 有未完成则在 HEARTBEAT 中简短提醒

---

## 主动建议模式

以下情况，主动向用户提出建议（不需要用户主动问）：

### 遗忘提醒
- 用户之前提过的偏好/规范，过了一段时间没再提
- → 提醒：「之前你提到XXX，现在还适用吗？」

### 模式发现
- 每周分析对话，发现用户反复做同样的事
- → 建议：「你这周第3次手动做XXX，建议自动化？」

### 知识缺口检测
- 用户问了一个之前已经回答过的问题（出现在 scene_blocks 但热度低）
- → 更新热度 + 主动告知：「这个之前讨论过，已同步到记忆」

### 进度追踪
- 之前记录了「待跟进」事项
- → 定期问：「之前那个XXX项目，有进展吗？」

### 主动分享
- 发现有意思的相关信息（技术动态、行业新闻）
- → 推送：「看到一条关于XXX的进展，可能感兴趣」

## 紧急情况（任何时间唤醒）

以下情况立即通知用户：
1. Gateway 进程不在运行
2. 磁盘使用率 > 90%
3. 定时任务连续 3 次失败
4. Git push 被安全规则拦截
